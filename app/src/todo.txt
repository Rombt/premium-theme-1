=============================  Для Envato необходимо добавить  ============================
   после того как тема принята на wordpress.org для Envato необходимо добавить
   Demo Importer
      Готовые демо (обычно несколько вариантов).
      Одно-кнопочный импорт (OCDI либо собственный импортер).
      Контент: XML, WIE (виджеты), JSON (Customizer).
      18. Трансляция#18 [skladchik.org].mp4
   Документация
      19. Трансляция#19 [skladchik.org].mp4  1:42:00


   1. Коммерческая структура темы (обязательное)

      WordPress.org принимает «базовые» темы.
      ThemeForest требует коммерческую упаковку:

      Demo Importer

      Готовые демо (обычно несколько вариантов).

      Одно-кнопочный импорт (OCDI либо собственный импортер).

      Контент: XML, WIE (виджеты), JSON (Customizer).

      Theme Options Panel

      Собственная панель настроек (Redux / Kirki / Customizer API).

      WordPress.org разрешает только Customizer — ThemeForest допускает полноценные options-панели.

      Собственные Widgets + Shortcodes

      Слайдеры, баннеры, иконки, info-box, portfolio-grid и т.д.

      Часто выносятся в плагин companion (требование Envato схоже с WordPress.org — функциональность ≠ тема).

      Customizer Pre-sets / Live Preview

      Готовые наборы настроек, pre-made styles.

      Header / Footer builder (часто желательно).

   2. Компаньон-плагин (обязательно)

      На WordPress.org большинство функций (Custom Post Types, Shortcodes, Widgets, Elementor widgets) запрещены внутри темы.
      На ThemeForest — допустимы, но желательно вынести в отдельный плагин, иначе получишь soft reject.

      В плагин выносят:

      CPT: Portfolio, Team, Testimonials, Services

      Metaboxes (ACF/MetaBox или собственные)

      Shortcodes

      Custom widgets

      Elementor blocks / Gutenberg blocks

      Demo importer файлы (если удобнее)

   3. Глубокая совместимость (только для Envato)

      Gutenberg compatibility

      Elementor compatibility

      WooCommerce deep integration

      Шаблоны магазина

      Custom product layouts

      Mini cart, Ajax cart

      RTL support (обязательно)

      Translation ready + .pot file

      Child theme (обязательно в комплекте)

      Cross-browser & responsive

      PHP 8.x compatibility

      Accessibility (A11y) минимум базовый

   4. Расширенный дизайн-пакет

      PSD/Figma исходники (часто требуют)

      Демо-сайт, готовый к просмотру

      Навигация между демо (demo switcher)

   5. Стандарты качества кода ThemeForest

      Соответствие Envato WordPress Theme Requirements

      Полный PHPCS:

      WordPress Core

      WordPress Docs

      WordPress VIP

      Скрипты и стили: минифицированные + версии

      Безопасность (sanitization/escaping)

      Производительность:

      Асинхронные скрипты

      Ленивая загрузка картинок

      Оптимизация запросов

      Полное отсутствие ошибок PHP/WP

      Полный HTML5/CSS3 валидатор

   6. Документация (строгий must-have)

      В комплекте должны быть:

      Полная HTML-документация

      Установка/обновление темы

      Импорт демо

      Настройки

      Частые вопросы

      Changelog

      Лицензии на все ассеты

   7. Пакет поставки (полностью обязательная структура)

      Обычно в архиве:

      yourtheme/
      ├── theme.zip (сама тема)
      ├── child-theme.zip
      ├── plugins/
      │   ├── yourtheme-core.zip
      │   ├── required-plugins.zip
      ├── documentation/
      │   └── index.html
      ├── demo-content/
      │   ├── content.xml
      │   ├── widgets.wie
      │   ├── customizer.json
      └── changelog.txt

   8. Дополнительные функции, без которых ThemeForest почти всегда отклоняет

      Mega Menu (часто ожидается)

      Ajax Search

      Page Builder Templates (Elementor/Block templates)

      Blog layouts (несколько вариантов)

      Portfolio layouts (много сеток)

      Header/Footer variations

   9. Лицензирование

      Envato требует:

      Все сторонние библиотеки должны быть GPL или совместимы.

      Font Awesome — ок.

      Google Fonts — ок.

      Иконки/картинки — собственные или свободные (Unsplash, Pexels).

      В документации — список лицензий.

   10. Что делать строго нельзя (причины reject)

      CPT/shortcodes внутри темы (желательно вынести в плагин).

      Демо-контент внутри темы.

      Жёсткое включение плагинов (вместо TGMPA).

      PHP предупреждения/Notice даже в dev.

      Inline CSS/JS без необходимости.

      Запросы к внешним API без явного согласия.
